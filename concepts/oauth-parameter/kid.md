[The name of the key (key id), which is an identifier generated by the resource server. It is RECOMMENDED that the authorization server generates this key id by computing a hash over the access_token, for example using SHA-1, and to encode it in a base64 format.](https://datatracker.ietf.org/doc/html/draft-ietf-oauth-v2-http-mac#section-4.1 "Read documentation for OAuth Parameter &#34;kid&#34;") (**[Internet Draft ietf-oauth-v2-http-mac: OAuth 2.0 Message Authentication Code (MAC) Tokens](/specs/IETF/I-D/ietf-oauth-v2-http-mac "This specification describes how to use MAC Tokens in HTTP requests to access OAuth 2.0 protected resources. An OAuth client willing to access a protected resource needs to demonstrate possession of a cryptographic key by using it with a keyed message digest function to the request. The document also defines a key distribution protocol for obtaining a fresh session key.")**)
